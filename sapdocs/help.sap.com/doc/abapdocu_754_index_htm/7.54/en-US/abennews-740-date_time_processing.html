Date/Time Processing in Release 7.40, SP02


1. Weaker type check for time zones



Modification 1

Weaker Type Check for Time Zones

A character-like data object is now sufficient in all operand positions where a time zone could previously only be specified with exactly the type TZNZONE or as a text field of the length 6. This concerns the statements CONVERT [INTO] TIME STAMP and formatting of string templates and WRITE.
