SELECT - Dynpro

Obsolete Syntax

SELECT *
       FROM dbtab
       WHERE col1 = f1 AND col2 = f2 ...
       INTO wa
       WHENEVER [NOT] FOUND
         SEND { ERRORMESSAGE | WARNING } [num [WITH { p1 ... }]].

Effect

The use of the statement SELECT as a standalone statement in the dynpro flow logic is obsolete. The effect of the statement is described in its (also obsolete) use as an addition of the statement FIELD .

Note

If the SELECT statement is used in the dynpro flow logic, table buffering is applied, if activated.
