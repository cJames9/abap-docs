DATA - tabkeys

Quick Reference

Syntax

... [ WITH key ]
    [ WITH secondary_key1 ] [ WITH secondary_key2 ] ...

Effect

Defines the table keys in an internal table. The following can be defined:

  • A primary table key using key.

    Every internal table has a primary key. The primary key must be defined explicitly for sorted tables and hashed tables. The information specified for the primary key can only be omitted for standard tables. The primary table key is then defined automatically as a non-unique standard key.

  • Multiple secondary table keys with secondary_key1, secondary_key2, ...

    Secondary keys are optional keys for all table categories. If no secondary key is specified, the internal table has only one primary key.

Example

Declares a hashed table with a unique primary key plus two non-unique secondary sorted keys, cityfrom_to and airp_from_to.

DATA spfli_tab TYPE HASHED TABLE OF spfli
  WITH UNIQUE KEY carrid connid
  WITH NON-UNIQUE SORTED KEY cityfrom_to  COMPONENTS cityfrom cityto
  WITH NON-UNIQUE SORTED KEY airp_from_to COMPONENTS airpfrom airpto.
