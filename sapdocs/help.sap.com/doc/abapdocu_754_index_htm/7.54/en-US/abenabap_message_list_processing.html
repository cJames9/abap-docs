Messages in List Processing

In list processing, a message of type "W" is always converted to type "E" before further context-dependent handling takes place. The message is then handled in accordance with the other context.

List processing is active in the following contexts with respect to messages:

  • In all executable programs called using SUBMIT, as long as no dynpro sequence called using CALL SCREEN is being processed there.
  • After list processing is called explicitly using LEAVE TO LIST-PROCESSING in a dynpro sequence. The statement LEAVE LIST-PROCESSING switches list processing off during a dynpro sequence.

In programs called using SUBMIT, list processing is active, therefore, apart from when actual list events are being handled (even when reporting events are being handled). This is especially true for the standard event START-OF-SELECTION.

Note

Background jobs are always started using SUBMIT, which means that list processing is usually also active at the same time.
