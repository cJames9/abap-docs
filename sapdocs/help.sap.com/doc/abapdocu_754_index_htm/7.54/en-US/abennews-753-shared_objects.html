Shared Objects in Release 7.53


1. Cross-server invalidations and releases



Modification 1

Cross-Server Invalidations and Releases

Until now, cross-server invalidations and releases of area instance versions were only possible for transactional areas. From Release 7.53, cross-server invalidations and releases can be performed for all areas with the Application Server area binding by using the associated parameter AFFECT_SERVER of the methods INVALIDATE_... and FREE_... of the area class in question. The parameter AFFECT_SERVER is added to the methods of existing area classes when a new generation is performed in transaction SHMA.
